setTimeout

public void setTimeout(int timeout)
Set the number of milliseconds after which to timeout API requests. This limit applies individually to both making a connection and waiting for a response, so technically (although quite unlikely) a request could persist for as long as twice the given timeout. Default is 30000, or 30 seconds. Setting this timeout will only have effect if done prior to using any of Chartboost's functionality.

getImpressionsUseActivities

Returns whether or not impressions will be displayed in a separate activity (true), or in a view at the front of the host activity (false).

setImpressionsUseActivities

public void setImpressionsUseActivities(boolean impressionsUseActivities)
Important: Call this method in your activity's onCreate() method.
Set whether or not to launch interstitials and the More Apps page in their own activity. This is often preferred for working with a NativeActivity. It may also be a more optimal solution for certain engines, such as Unity. Please use care, as the onPause-onStop/onStart-onResume cycle will be called for the originating activity when showing and hiding the impressions. If false (default behavior), impressions will be displayed in a view above the other content views in your host activity.
